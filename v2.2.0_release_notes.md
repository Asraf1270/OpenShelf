# 🚀 Release Notes: OpenShelf v2.2.0 — The Complete Experience

**Release Date:** April 23, 2026

We are excited to announce the release of **OpenShelf v2.2.0**, a comprehensive update that brings powerful new user-facing features, a fully modernized admin panel, and a restored email notification pipeline. This release bridges the gap between backend strength and frontend polish, delivering a complete, production-ready library management experience.

---

### 📖 New Feature: My Borrowed Books

Users now have a dedicated **My Borrowed** page to stay on top of their reading:

- **Active Borrow Tracking:** View all currently borrowed books with cover art, title, and owner details at a glance.
- **Visual Due-Date Progress:** An intuitive progress bar shows how much borrowing time remains, with color-coded urgency indicators (green → yellow → red).
- **One-Click Returns:** Initiate the return process directly from the borrowed books page — no more navigating through multiple menus.
- **Mobile-First Design:** Fully responsive card layout with native dark mode support.

---

### 🔑 New Feature: Forget Password

A secure, self-service password recovery flow has been added:

- **Two-Factor Verification:** Users must provide both their registered **email** and **phone number** to initiate recovery, preventing unauthorized resets.
- **OTP via Email:** A time-sensitive verification code is sent to the user's email using the OpenShelf email system.
- **Seamless Reset:** After successful verification, users can set a new password and are redirected to login immediately.
- **Branded Email Template:** A dedicated `emails/forget_password.php` template ensures a consistent, professional notification experience.

---

### 🎨 Admin Panel Modernization

The entire administrative interface has been redesigned from the ground up:

- **Mobile-First Responsive Layout:** Every admin page — dashboard, users, books, requests, announcements — is now fully optimized for smartphones and tablets.
- **Card-Based Design:** Replaced legacy table-heavy layouts with modern, scannable card grids.
- **Full Dark Mode Compatibility:** The admin panel seamlessly integrates with OpenShelf's global theme engine, including charts, modals, and data grids.
- **Centralized Admin Styles:** Introduced a unified `admin.css` design system for visual consistency across all management modules.
- **Enhanced User & Book Management:** Improved data grids with better search, filtering, and mobile-friendly bulk action bars.

---

### 📧 Email Notification System Restoration

The end-to-end email notification pipeline has been fully restored and hardened:

- **Borrow Request Alerts:** Book owners are notified instantly when someone requests to borrow their book.
- **Approval & Rejection Notices:** Borrowers receive clear confirmation emails when their requests are approved or declined.
- **Return Notifications:** Owners are informed when a borrowed book has been returned.
- **Mailer Configuration Fix:** Resolved initialization issues in the `Mailer` class and `config/mail.php` to ensure reliable SMTP delivery across all modules.

---

### 📢 Navigation & Accessibility

- **Announcements in Navigation:** Added dedicated "Announcement" links to both the desktop navbar and mobile hamburger menu, ensuring community updates are always one click away.
- **Footer Link Corrections:** Updated all footer navigation links to use correct `.php` extensions.

---

### 🐛 Bug Fixes

- **Announcements Page Crash:** Fixed a 500 Internal Server Error on `/announcements/index.php` caused by a code-level issue.
- **Admin Books Management:** Resolved errors in `/admin/books/index.php` that prevented proper book listing and management.
- **Add Book Database Connection:** Fixed missing database initialization on the add-book page that caused submission failures.
- **Login Remember Me:** Corrected the "Remember Me" checkbox functionality on the login page.
- **Registration Flow:** Improved error handling and form processing in the registration module.
- **Return Book Processing:** Enhanced the return-book workflow with better error handling and email notification integration.

---

### 🛠️ Technical Improvements

- **CSS Variable Integration:** Migrated hardcoded admin colors to CSS custom properties for easier theming and maintenance.
- **Mailer Class Hardening:** Improved `lib/Mailer.php` with better error handling, configuration validation, and debug logging.
- **Responsive CSS Architecture:** Replaced legacy float-based layouts with modern CSS Grid and Flexbox across admin modules.
- **SVG Iconography:** Standardized on FontAwesome 6.5.1 and custom SVGs for crisp visuals on all display resolutions.

---

### 📦 How to Upgrade

1. **Backup:** Create a full backup of your database and `/uploads` directory before proceeding.
2. **Deploy:** Replace all project files with the latest version from the repository.
3. **Database:** No schema changes required — v2.2.0 is fully compatible with the v2.0.0 database schema.
4. **Environment:** No `.env` changes required.
5. **Cache:** Clear your browser cache to ensure the new styles and scripts are loaded correctly.
