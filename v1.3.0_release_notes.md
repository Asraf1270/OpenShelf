# OpenShelf v1.3.0: Performance & Data Optimization Update

We are excited to release **OpenShelf v1.3.0**! This update focuses on significant backend improvements to enhance system performance, data management efficiency, and scalability. We've restructured the notification system for better performance and implemented intelligent data limiting to keep the platform running smoothly.

## 🌟 Release Title
**OpenShelf v1.3.0: Performance & Data Optimization Update**

## 🚀 Key Highlights

### 📊 Notification System Overhaul
- **Per-User Storage Migration:** Completely restructured notification storage from a single global file to individual user-specific JSON files, eliminating the need to scan through all notifications for each user.
- **Smart Data Limiting:** Implemented automatic limiting to store only the 25 most recent notifications per user, preventing data bloat and maintaining optimal performance.
- **Lent History Optimization:** Added similar intelligent limiting to the lent history records, keeping only the 25 most recent entries per user for streamlined data management.

### ⚡ Performance Enhancements
- **Reduced I/O Operations:** Eliminated global file scanning for notifications, resulting in faster load times and reduced server resource usage.
- **Scalable Architecture:** The new per-user storage approach ensures the system can handle growing user bases without performance degradation.
- **Memory Efficiency:** Smaller data files per user mean faster parsing and lower memory consumption during operations.

### 🛠️ Backend Refactoring
- **Codebase Modernization:** Updated all notification-related functions across the application to work seamlessly with the new storage system.
- **Backward Compatibility:** Maintained full compatibility with existing user data while improving the underlying architecture.
- **Data Integrity:** Ensured all existing notifications and lent history are properly migrated and preserved during the transition.

### 🔧 System Improvements
- **Automated Data Management:** The system now automatically sorts and limits notification and history data, ensuring consistent performance over time.
- **Enhanced Reliability:** Reduced potential for data corruption by isolating user data into separate, manageable files.
- **Future-Proof Design:** Laid the groundwork for additional per-user features and optimizations.
