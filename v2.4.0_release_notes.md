# 📦 OpenShelf v2.4.0 Release Notes
## 🚀 v2.4.0 — Aesthetic Overhaul & Community Safety
**Release Date:** April 24, 2026

Version 2.4.0 is a major milestone in the visual evolution of OpenShelf. This release focuses on delivering a premium, modern experience for both new and existing users, while introducing critical safety and community-building features.

---

### 🎨 Key Highlights

#### 🏠 Reimagined Landing Page
The entry point to OpenShelf has been completely rebuilt from the ground up.
- **Mission-First Design**: Focuses on explaining the "Why" and "How" of OpenShelf to new visitors.
- **Modern Aesthetics**: Utilizes a premium indigo-gradient design with high-fidelity typography (Inter).
- **Interactive Stats**: Real-time counters showing total books, active readers, and availability.
- **Mobile-First**: Perfectly optimized for seamless scrolling on smartphones and tablets.

#### 🚩 Integrated Reporting System
Safety is our priority. We've introduced a dedicated **Report Issue** page.
- **Centralized Feedback**: Users can now report technical bugs, user misconduct, or data inaccuracies.
- **Direct Admin Feed**: Reports are captured and stored for immediate administrative review.
- **Smart Forms**: Auto-populates user data for authenticated members to streamline the process.

#### 🌗 Global Info Page Modernization
We've refreshed every informational page to match our v2.4 design language.
- **Full Theme Support**: `About`, `Contact`, `FAQ`, `Privacy`, and `Terms` now fully support native **Light** and **Dark** modes.
- **Accordion FAQ**: Interactive Q&A interface for faster information retrieval.
- **Contact Grid**: Clean, icon-supported contact info with direct WhatsApp integration links.
- **Policy Cards**: Simplified privacy and terms layouts using structured cards and iconography.

#### 📚 Community Growth Policy
To ensure OpenShelf remains a vibrant resource, we've updated our **Terms of Service**:
- **The 2-Book Rule**: New users are now required to list at least 2 books for sharing within their first 30 days. This ensures every member contributes to the collective knowledge of the campus.

---

### 🛠️ Technical Improvements
- **CSS Variables**: Transitioned all info pages to a unified CSS variable system for consistent theming.
- **Animation Suite**: Integrated subtle micro-animations (fade-up, hover scale) to enhance the feeling of a "Premium" app.
- **Responsive Grids**: Replaced legacy layouts with modern CSS Flexbox and Grid systems.

---

### 📦 Installation & Setup
1. Pull the latest changes from the repository.
2. Ensure the `data/reports/` directory is writable by the web server.
3. No database migrations are required for this update.
