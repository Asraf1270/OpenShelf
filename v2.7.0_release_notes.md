# OpenShelf v2.7.0 Release Notes
**The Notification & UX Evolution**
**Release Date:** April 29, 2026

## 📝 Overview
OpenShelf v2.7.0 is a major release focused on stabilizing the platform's core communication infrastructure and simplifying the user onboarding journey. By centralizing the notification system and removing administrative bottlenecks, we've made the platform more reliable and user-friendly than ever before.

## 🚀 Major Features

### 📧 Unified Mailer Engine
We have completely refactored how OpenShelf communicates with its users. All emails are now handled by a centralized `Mailer` library that ensures:
- **Consistent Branding:** Every email looks like it belongs to the same premium platform.
- **Improved Reliability:** Optimized SMTP handling via modern libraries.
- **Easy Maintenance:** A single point of control for all system messages.

### ⚡ Streamlined Onboarding
To encourage faster community growth, we've removed the manual "Admin Approval" step for new users. 
- **Immediate Access:** Users can now browse, share, and borrow books as soon as they verify their email via OTP.
- **Automated Safety:** Domain-locked registration and OTP verification continue to provide robust protection against bots.

### 🛡️ Enterprise-Grade Template Stability
A core focus of this release was eliminating the "Undefined Variable" errors that previously plagued the email system.
- **Robust Fallbacks:** All templates now include safe defaults and `isset()` checks for every variable.
- **Graceful Failures:** If data is missing (e.g., a missing due date), the email still renders beautifully with logical placeholders instead of crashing.

## 🛠️ Technical Changes
- **Controller Alignment:** Updated `admin/requests/index.php` and other controllers to pass standardized data keys (`borrower_name`, `due_date`, etc.) to the mailer.
- **URL Safety:** Implemented global `$base_url` fallbacks in all email templates to ensure links always work, regardless of the calling context.
- **Design Refresh:** Applied a clean, mobile-responsive layout to all 12+ system email templates.

## 🐛 Bug Fixes
- Fixed `Undefined variable $base_url` in `request_approved.php`.
- Fixed `Undefined variable $announcement_priority` in `announcement.php`.
- Fixed `Undefined variable $return_date` in `book_returned.php`.
- Resolved inconsistent naming conventions between user and borrower name variables in notifications.
